# Synthetic Data Statement

All telemetry in this repository is synthetic.

It is not customer data, victim data, production data, or evidence from a real incident.

The scenarios are designed to model realistic investigation patterns for education, detection engineering, SOC workflow discussion, and SIEM lab demonstrations.

## Intent

The goal is to help defenders practice and discuss what should happen after a detection fires:

- what context matters
- what pivots should come next
- what supporting evidence should be reviewed
- what should be documented
- what would justify escalation

## Source basis

Some labs may be inspired by public threat behavior, public reporting, vendor research, CISA advisories, MITRE ATT&CK techniques, or common adversary tradecraft.

When that happens, the lab should clearly distinguish between:

- confirmed public facts
- behavior patterns being modeled
- assumptions made for the synthetic scenario
- telemetry invented for the lab

## Usage guidance

Do not treat these labs as authoritative threat intelligence.

Do not treat the mock telemetry as proof of real adversary behavior.

Use the labs to reason about detection coverage, investigation workflow, SIEM data requirements, analyst pivots, and response notes.
